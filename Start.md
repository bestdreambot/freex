🎯 Цель проекта

Создать удобный мобильный ИИ-стенд, чтобы экспериментировать с оркестрацией моделей (routing, multi-agent) прямо с телефона в дороге, без необходимости сидеть за компьютером.

---

🚀 Что такое FreeX

FreeX — это личный мобильный лабораторный стенд для работы с несколькими AI-моделями одновременно. Позволяет отправлять запрос параллельно к DeepSeek, Grok, Gemini, Qwen и другим моделям, сравнивать ответы и выбирать лучший.

Основан на идее OpenFugu (умный оркестратор, который выбирает лучшую модель для задачи).

✅ Что уже сделано (по состоянию на 29.06.2026)

1. Инфраструктура

· Создан репозиторий bestdreambot/freex
· Развёрнут на Vercel (freex-xi.vercel.app)
· Переход на единый index.html с разделением на 3 файла (HTML + CSS + JS)

2. Мобильный интерфейс (Grok-стиль)

· Адаптивный дизайн под iPhone и Android
· Левое боковое меню (как в DeepSeek)
· История переписки сохраняется в localStorage
· Папки для группировки чатов (создание/переименование/удаление)
· Поиск по чатам
· Тёмная тема, фикс поля ввода (больше не обрезается)

3. Поддержка AI-моделей

Модель API Доступ
DeepSeek DeepSeek API ✅ (требуется ключ)
Gemini Google AI Studio ✅ (ограниченно)
Groq Groq Cloud ✅ (быстро)
Qwen Alibaba DashScope ✅ (бесплатный)
Grok (Free) OpenRouter (x-ai/grok-4-fast:free) ✅ (промо-период)
Kimi (Free) OpenRouter (moonshotai/kimi-k2.7:free) ✅ (промо-период)

4. Multi-Agent режим

· Запрос уходит к нескольким моделям одновременно (параллельно через Promise.all())
· Можно выбрать 2+ модели через чекбоксы в интерфейсе
· Ответы отображаются с подписями: «🤖 DeepSeek», «🤖 Grok» и т.д.

5. Безопасность

· API ключи хранятся только в localStorage — не передаются на сервер
· Нет хранения ключей на GitHub
· Ключи вводятся через интерфейс настроек

6. Управление данными

· Все чаты и папки хранятся в localStorage
· Кнопка «Экспорт» для скачивания всех данных в JSON
· При очистке кэша данные теряются (нужен экспорт)
