# RESTORE.md — восстановление FreeX Development Machine после форматирования

Пошаговая инструкция «с нуля» после чистой установки Windows на этот же
ноутбук (или его замену). Каждый шаг: что установить → откуда → что
настроить → как проверить → что делать при ошибке. Секретов в этом файле
нет и быть не должно — см. `SECRETS_CHECKLIST.md` для того, откуда их
вернуть.

Точная модель для драйверов и BIOS: **ASUS VivoBook X512FL / X512FLC**
(подтверждено `Win32_ComputerSystem`/`Win32_BaseBoard`).

---

## 1. Установка Windows

- Установить: Windows 11 Pro (та же редакция, что была).
- Откуда: официальный образ Media Creation Tool с сайта Microsoft.
- Настроить: локальная учётная запись или Microsoft-аккаунт — по
  усмотрению, диск для установки — тот же SSD (Intel SSDPEKNW512G8, 512 ГБ).
- Проверка: `winver` показывает Windows 11 Pro, `Get-CimInstance
  Win32_OperatingSystem` не выдаёт ошибок.
- При ошибке: проверить загрузочную флешку (переписать заново через Rufus),
  проверить, что в BIOS выбран правильный boot-режим (UEFI, не Legacy).

## 2. Активация Windows

- Настроить: `Параметры → Система → Активация`.
- Проверка: статус должен быть «Windows активирована с помощью
  цифровой лицензии». **Важно**: на момент этого паспорта проверка через
  `SoftwareLicensingProduct.LicenseStatus` вернула код **5** (не «1 —
  Licensed»), это статус уведомления/грейс-периода, а не подтверждённая
  активация — перед форматированием стоит вручную открыть
  `Параметры → Активация` и убедиться, что там написано на самом деле,
  а не полагаться на автоматическую проверку.
- При ошибке: цифровая лицензия обычно привязана к железу/Microsoft-аккаунту
  и должна подхватиться сама при переустановке на этом же ноутбуке; если нет
  — `Устранение неполадок` в разделе активации.

## 3. Windows Update

- Настроить: поставить все обновления, перезагрузиться, повторить, пока
  не останется «Вы в курсе последних событий».
- Проверка: `Get-CimInstance Win32_QuickFixEngineering | Sort InstalledOn
  -Descending | Select -First 5`.
- При ошибке: `Устранение неполадок → Центр обновления Windows`
  (встроенный troubleshooter).

## 4. Драйверы ASUS

- Установить: чипсет, аудио (Realtek), Wi-Fi (Realtek 8822CE), тачпад
  (ASUS Precision Touchpad / ELAN), видео — Intel UHD Graphics + NVIDIA
  GeForce MX250 (Optimus).
- Откуда: **support.asus.com**, поиск по точной модели
  `VivoBook X512FL / X512FLC` — не ставить драйверы «на глаз» с сайтов
  NVIDIA/Intel напрямую для GPU, у ASUS свои OEM-сборки под Optimus,
  универсальные иногда конфликтуют.
- Настроить: после установки — `NVIDIA Control Panel` → назначить Intel UHD
  графикой по умолчанию для VS Code/Chrome, MX250 — только по запросу
  (экономия энергии, см. `comp.md`).
- Проверка: `Get-CimInstance Win32_VideoController | Select Name,
  DriverVersion, DriverDate` — обе даты должны быть свежее прежних
  (Intel — было 05.02.2021, NVIDIA — было 01.02.2022).
- При ошибке: если Wi-Fi не поднялся сразу — драйвер Realtek 8822CE ставить
  первым делом отдельно, до остальных, иначе интернета для скачивания
  остального не будет (использовать Ethernet/телефон как временный источник
  для первого драйвера).

## 5. Git

- Установить: Git for Windows.
- Откуда: **git-scm.com** (была версия 2.51.0.windows.1).
- Настроить: `git config --local user.name` / `user.email` — **локально
  на уровне репозитория FreeX**, не глобально (осознанное решение проекта,
  см. `agents/gpt.md`); явно задать `core.autocrlf` при первом клоне
  (см. `comp.md` §6), чтобы не ловить предупреждения про CRLF/LF.
- Проверка: `git --version`, `git config --local --list` в клонированном
  `freex` показывает user.name/user.email.
- При ошибке: «Author identity unknown» при коммите — значит забыли задать
  `user.name`/`user.email` локально, это уже случалось один раз в истории
  проекта.

## 6. PowerShell

- Уже встроен в Windows 11 (Windows PowerShell 5.1, было `5.1.26100.8655`).
- Настроить: ничего специального не требуется — все скрипты FreeX
  запускаются с `-ExecutionPolicy Bypass -File`, глобальную политику
  выполнения менять не нужно.
- Проверка: `$PSVersionTable.PSVersion` выводит версию без ошибок.
- При ошибке: если скрипты `.ps1` отказываются запускаться совсем —
  проверить не Constrained Language Mode ли включён политикой безопасности.

## 7. VS Code

- Установить: Visual Studio Code (была версия 1.127.0).
- Откуда: **code.visualstudio.com**.
- Настроить: поставить **только** два расширения —
  `anthropic.claude-code` и `ms-vscode.powershell`. Применить
  `settings.json` из `comp.md` §6 (`files.watcherExclude`,
  `telemetry.telemetryLevel: off`). Не ставить лишние расширения без
  повода (см. правило «зачем это FreeX» в `comp.md`).
- Проверка: `code --version`, `code --list-extensions` показывает ровно
  два расширения.
- При ошибке: если `code` не виден в PATH — переустановить с галочкой
  «Add to PATH» на этапе установки.

## 8. Claude Code

- Установить: Claude Code CLI (`npm install -g @anthropic-ai/claude-code`
  или официальный установщик — проверить актуальный способ на
  claude.com/code на момент восстановления, это быстро меняется).
- Настроить: авторизация через `claude auth` (или `setup-token`) —
  **без сохранения токена в файлах проекта**. Расширение для VS Code
  ставится автоматически или из Marketplace.
- Проверка: `claude --version`, `claude --help` отрабатывают; в VS Code
  видна панель Claude Code.
- При ошибке: если CLI не находится в PATH после установки — перезапустить
  терминал/VS Code, проверить `where claude` (Windows) / `which claude`.

## 9. RustDesk

- Установить: RustDesk (была версия 1.4.9+67).
- Откуда: **rustdesk.com** (официальный сайт, не сторонние сборки).
- Настроить: включить unattended access (постоянный пароль) **только если
  реально нужен удалённый доступ с телефона регулярно** — иначе не ставить
  вообще (см. `comp.md`, кандидат на отключение автозапуска). Конфиг лежит
  в `%APPDATA%\RustDesk\config\` — ID и пароль генерируются заново при
  установке, из GitHub их брать неоткуда и не нужно (в репозиторий они
  никогда не попадали).
- Проверка: подключиться с телефона по новому ID, убедиться, что видно
  экран.
- При ошибке: служба `RustDesk` не стартует — см. известный баг из этого
  паспорта (30-секундный таймаут при загрузке), проверить `Get-Service
  RustDesk` вручную запустить, если не поднялась сама.

## 10. Браузер

- Установить: Google Chrome (единственный браузер — см. правило
  «один браузер» в `comp.md`, не ставить Edge/Yandex без явной причины).
- Откуда: **google.com/chrome**.
- Настроить: войти в Google-аккаунт для синхронизации закладок/паролей
  (см. `backup/PERSONAL_FILES_CHECKLIST.md` — экспорт закладок делается
  ДО форматирования); включить Memory Saver
  (`chrome://settings/performance`).
- Проверка: открыть `freex-eta.vercel.app`, убедиться, что сайт грузится.
- При ошибке: закладки не синхронизировались — восстановить из
  экспортированного HTML-файла (раздел 6 личных файлов).

## 11. Клонирование FreeX с GitHub

```
git clone https://github.com/bestdreambot/freex.git
```

- Настроить: клонировать **один раз, в одну папку** (в прошлый раз
  образовался забытый дубль в `C:\Users\Андрей\Freex\` — не повторять,
  клонировать сразу в `C:\Users\Андрей\Freex\freex\` и больше нигде).
- Проверка: `git remote -v` — URL **без** встроенного токена
  (`https://github.com/bestdreambot/freex.git`, не
  `https://oauth2:TOKEN@...`).
- При ошибке: просит логин/пароль при клонировании — авторизоваться через
  `gh auth login` или Git Credential Manager, не вставлять токен в URL.

## 12. Восстановление переменных окружения

- Настроить: создать `.env.local` в корне `freex` (файл в `.gitignore`,
  из GitHub не восстановится — это ожидаемо и правильно).
- Значения — см. `SECRETS_CHECKLIST.md`, откуда взять каждый ключ
  (`DEEPSEEK_API_KEY` и т.д.). Для продакшена — эти же переменные уже
  должны быть настроены в Vercel Environment Variables (не теряются при
  переустановке компьютера, они на сервере Vercel, не локально).
- Проверка: локальный `vercel dev` (если используется) подхватывает ключ,
  DeepSeek отвечает без ошибки авторизации.
- При ошибке: 401/403 от DeepSeek — значит `.env.local` не создан или ключ
  вставлен неверно.

## 13. Настройка Git identity

- Настроить (повтор шага 5, отдельно вынесено, т.к. критично):
  `git config --local user.name "Андрей"` и
  `git config --local user.email "<используемый email>"` внутри
  клонированной папки `freex`.
- Проверка: `git commit --allow-empty -m "test"` не выдаёт «Author identity
  unknown» (после проверки — `git reset HEAD~1`, тестовый коммит не нужен
  в истории).
- При ошибке: см. шаг 5.

## 14. Запуск FreeX

- Проверка структуры: `powershell -File scripts/resume-freex.ps1` из
  корня `freex` — должен отработать без ошибок и показать состояние
  проекта.
- Открыть `index.html` локально или поднять `vercel dev` для полного
  теста с `api/chat.js`.
- При ошибке: см. вывод самого `resume-freex.ps1` — он явно показывает,
  какие ключевые документы не найдены.

## 15. Проверка сайта и API

- Открыть `https://freex-eta.vercel.app` в браузере.
- Отправить тестовое сообщение в чат — должен ответить DeepSeek (работает
  даже без личного ключа, сервер использует свой `DEEPSEEK_API_KEY` из
  Vercel).
- При ошибке: смотреть `vercel logs` в проекте на Vercel, проверить, что
  переменные окружения не потерялись на самом Vercel (они не зависят от
  этого компьютера).

## 16. Проверка команды «ок»

- Написать в новом чате с Claude Code одно слово: `ок`.
- Ожидаемое поведение (протокол в `AI_RESUME.md`): агент читает
  `AI_RESUME.md`/`PROJECT_STATUS.md`, сообщает текущую задачу и последний
  шаг, ничего не меняет без отдельной команды.
- При ошибке: если агент не в курсе протокола — убедиться, что открыт
  правильно склонированный `freex` (а не старая папка), и что
  `AI_RESUME.md` реально существует и не пустой.

## 17. Проверка remote control

- Подключиться к этому ноутбуку с телефона через RustDesk по новому ID.
- Проверка: видно экран, можно управлять курсором.
- При ошибке: см. шаг 9 — служба RustDesk должна быть в состоянии
  `Running` (`Get-Service RustDesk`).

## 18. Финальная диагностика

Прогнать по очереди:
```
powershell -File scripts/resume-freex.ps1
powershell -File scripts/checkpoint-freex.ps1
```
Оба должны отработать без ошибок: git виден, версия FreeX определена,
`node --check` проходит по всем `.js`, секретов не найдено. Дополнительно
пройтись по чек-листу из `comp.md` §10 «Как проверить, что всё готово».
