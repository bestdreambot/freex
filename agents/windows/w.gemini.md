1. Граница очистки: Что допустимо, а что — табу

3. Что НЕЛЬЗЯ удалять: Столпы стабильности
Некоторые компоненты кажутся лишними, но их удаление ломает внутренние связи Windows (COM-объекты, зависимости Win32).
Обязательно оставить (Трогать запрещено):
 * Microsoft Store, App Installer, winget: Главные инструменты установки и обновления софта (включая Terminal, Git и VS Code) без поиска по сомнительным сайтам.
 * Edge WebView2 Runtime: Критически важный системный движок. Без него не запустятся многие современные программы, включая VS Code (частично использует его для рендеринга некоторых панелей), Discord, RustDesk и кучу современных лаунчеров.
 * Windows Security (Defender, Firewall, SmartScreen): Без них машина с API-ключами FreeX и npm-пакетами станет проходным двором. Defender в Windows 11 отлично оптимизирован и в простое потребляет минимум ресурсов.
 * Windows Recovery Environment (WinRE) & Windows Update: Без обновлений безопасности машина разработчика — цель №1. Без WinRE при любом сбое диска ты потеряешь доступ к системе без возможности восстановления.
 * BITS (Background Intelligent Transfer Service): Необходим не только для Windows Update, но и для загрузки пакетов через winget и синхронизации фоновых служб.
 * Notepad, Snipping Tool, Calculator: Легковесные нативные утилиты. Snipping Tool необходим для фиксации багов и UI-ошибок FreeX.
Можно удалить при конкретном условии:
 * Microsoft Edge: Удалить его полностью технически сложно и неразумно (сломает PDF-просмотрщик по умолчанию и системные вызовы браузера). Решение: Оставить его как резервный браузер для тестирования кроссбраузерности FreeX, но жестко пресечь его фоновую активность (Startup Boost).
4. Автозагрузка: Жесткая гигиена
Каждый процесс в автозагрузке отъедает драгоценную RAM объемом 8 ГБ.
Разница между типами автозапуска:
 * Приложение в автозагрузке (Startup App): Запускается в контексте пользователя при входе в систему. Легко отключается в Диспетчере задач.
 * Служба (Service): Работает на уровне системы, часто с правами SYSTEM. Может запускаться до входа пользователя.
 * Запланированная задача (Scheduled Task): Запускается по триггеру (время, событие, простой). Коварна тем, что может нагрузить CPU прямо во время созвона или сборки проекта.
 * Фоновый процесс браузера: Edge или Chrome продолжают висеть в памяти после закрытия окна, чтобы «быстрее открываться».
Схема автозагрузки для FreeX Development Machine:
 * Должно запускаться автоматически:
   * Windows Security Notification Icon (системный трей безопасности).
   * Менеджер паролей (Bitwarden или аналогичный) — безопасность ключей превыше всего.
   * RustDesk (только если нужен постоянный удаленный доступ с телефона, иначе запускать вручную).
   * Аудиодрайверы (Realtek/Waves - часто необходимы для работы микрофона на созвонах).
 * Запускать строго вручную:
   * VS Code, Kilo Code, Claude Code: Не должны висеть в памяти, пока ты не сел кодить.
   * Git / GitHub Desktop: Запускаются по требованию.
   * Браузеры (Chrome, Edge): Отключить опцию «Разрешить работу служб в фоновом режиме при закрытии браузера» в настройках самих браузеров.
   * ASUS/Intel-утилиты (Armoury Crate, MyASUS, Intel Command Center): Главные пожиратели ресурсов. Удалить или отключить их службы. Профили вентиляторов настраиваются один раз в BIOS.
5. Службы Windows: Отключаем только то, что не убьет систему
Отключение служб ради «оптимизации» — миф из эпохи Windows XP с 256 МБ RAM. В Windows 11 неиспользуемые службы находятся в состоянии Вручную (Запуск по требованию) и потребляют 0% CPU и 0 МБ RAM, так как их код не загружен в память. Но некоторые «вредные» службы все же стоит перевести в режим Отключена.
Можно отключить безболезненно (если функции не используются):
 * Служба географического положения (lfsvc): Если не нужны карты на ПК. (Освобождает ~2 МБ RAM).
 * Диспетчер печати (Spooler): Если к ПК не подключен физический принтер. (Освобождает ~5 МБ RAM).
 * Удаленный реестр (RemoteRegistry): Критическая дыра в безопасности, не нужна на рабочей машине. (0 МБ в простое, но повышает защиту).
 * Службы Xbox (XblAuthManager, XblGameSave, XboxNetApiSvc): Если Xbox-приложение удалено. (Освобождает ~10 МБ RAM).
Не трогать (Даже если советуют в интернете):
 * Служба системы событий COM+ (EventSystem) & Теневое копирование тома (VSS): Сломает создание точек восстановления и резервное копирование.
 * Служба времени Windows (W32Time): Если отключить, собьются часы. Git и API-провайдеры (OpenAI, GitHub, Anthropic) будут отклонять твои HTTPS-запросы из-за несовпадения таймштампов сертификатов SSL.
 * Службы шифрования (CryptSvc): Без нее перестанут работать любые безопасные веб-соединения и проверка цифровых подписей драйверов.
6. Процессы, которые выглядят как мусор, но являются нормальными
При открытии Диспетчера задач на 8 ГБ RAM может начаться паника. Давай разберемся, кто есть кто:
 * svchost.exe (Service Host): Это системный контейнер. Сама по себе эта программа — просто оболочка для запуска DLL-служб. В Windows 11 их может быть больше 80 штук. Это нормально. Windows группирует их, чтобы падение одной службы не роняло всю систему.
 * Runtime Broker (RuntimeBroker.exe): Посредник между классической Windows и UWP-приложениями (проверяет разрешения на доступ к камере, микрофону и файлам). Если он потребляет более 15 МБ RAM — значит, какое-то фоновое UWP-приложение ведет себя некорректно.
 * SearchHost.exe / StartMenuExperienceHost.exe: Отвечают за работу поиска и меню «Пуск». Они приостанавливают свою работу (впадают в «спячку»), когда ты их не используешь.
 * Antimalware Service Executable (MsMpEng.exe): Это сам Защитник Windows. Он может временно нагружать CPU до 100% при первом запуске компьютера (сканирование автозагрузки) или при установке npm-пакетов (когда в папку node_modules залетает 10 000 мелких файлов). Это плата за безопасность. Не нужно пытаться его «убить».

Честная сравнительная оценка:
Самый дешевый и эффективный способ ускорить FreeX Development Machine — добавить планку памяти до 16 ГБ. На 8 ГБ запуск VS Code + Edge (с локальным FreeX) + Node.js-сервер + один AI-агент в фоне сразу забьют память на 85%. Система уйдет в файл подкачки (swapping), что снизит скорость диска SSD и общую отзывчивость.
Если физический апгрейд невозможен: твой главный враг — Electron-приложения и вкладки браузера, а не службы Windows. Держи запущенными только VS Code и одну вкладку Edge/Chrome с тестируемым сайтом.
8. Безопасность чистой машины разработчика
Поскольку ты работаешь с приватными ключами API и деплоем в GitHub, безопасность не может быть принесена в жертву скорости.
Что должно быть включено всегда:
 * Windows Defender + SmartScreen: SmartScreen предотвратит запуск случайно скачанного вредоносного бинарника. Defender защитит от троянов, ворующих сессии браузера (куки с авторизацией GitHub).
 * UAC (User Account Control): Держи ползунок на уровне по умолчанию. Он не дает скриптам незаметно получить права администратора и пропатчить файлы Node.js или VS Code.
 * Защита от потенциально нежелательных приложений (PUA): Блокирует скрытую установку рекламного софта и майнеров, которые часто идут в комплекте с бесплатными утилитами.
Нужен ли сторонний антивирус?
Категорически нет. Сторонние антивирусы (Kaspersky, Avast, McAfee) внедряют свои сертификаты в браузеры, перехватывают трафик, жрут от 200 до 500 МБ RAM и часто конфликтуют с локальными серверами Node.js, блокируя порты localhost:3000 или localhost:5000. Встроенного Defender более чем достаточно.
9. Критический обзор инструментов очистки из интернета
Использование твикеров на рабочей машине — русская рулетка.
 * Win11Debloat / Sophia Script / Chris Titus Utility:
   * Что делают: Автоматизируют удаление UWP, отключение телеметрии, настройку интерфейса.
   * Вердикт: Chris Titus Tech Utility — самый прозрачный инструмент (работает на чистом PowerShell, код открыт, можно запускать отдельные функции). Sophia Script — мощный, но слишком агрессивный для новичков.
   * Подходят ли для FreeX: Только для ознакомления и точечного копирования команд. Запуск "всего скрипта целиком" гарантированно сломает какую-нибудь важную функцию (например, синхронизацию времени или работу WSL).
 * O&O ShutUp10++:
   * Что делает: Отключает телеметрию и настройки приватности Microsoft.
   * Вердикт: Самая безопасная утилита в классе. Есть четкие маркеры "Рекомендуется" (безопасно) и предупреждения. Позволяет легко откатить изменения одной кнопкой. Рекомендуется для отключения рекламы и базового шпионажа MS.
 * Tiny11 / AtlasOS / ReviOS:
   * Что делают: Модифицированные дистрибутивы Windows.
   * Вердикт: Опасны. В них вырезаны механизмы безопасности, центр обновлений и системные библиотеки. Использовать их на машине, где лежат API-ключи от твоих платных аккаунтов — критический риск.
10. Практика опытных пользователей (Разработчики & Админы)
Специалисты, работающие на Windows с 8 ГБ RAM, давно пришли к единому консенсусу:
 * Что они удаляют: OneDrive, Teams, Widget-панель (Widgets ломают производительность Explorer.exe), встроенные игры.
 * Что они отключают: Эффекты прозрачности Windows 11 (разгружает встроенную графику Intel/AMD на слабых процессорах), Startup Boost в Edge, индексацию поиска Windows Search для тяжелых папок проектов (например, папки node_modules заносятся в исключения индексации, иначе диск будет нагружен на 100%).
 * Что они оставляют: Центр обновлений Windows и защитник Defender. Безопасность и стабильность важнее 50 МБ сохраненной памяти.

13. Измерения «До» и «После»
Показатели, на которые мы должны смотреть, чтобы понять, успешна ли очистка:
 * Число процессов в Диспетчере задач (Performance -> CPU): На чистой Windows 11 после загрузки их около 130-150. Наша цель — уложиться в менее 110 процессов в простое.
 * Использование RAM через 5 минут после старта (в простое): На 8 ГБ RAM чистая Windows 11 забирает около 3.2 - 3.8 ГБ. Наша цель после оптимизации — менее 2.8 ГБ.
 * CPU в простое: Должен колебаться в пределах 0–2%. Если постоянно висит 5–10% — значит, какая-то служба (например, поиск или виджеты) работает вхолостую.
14. План отката
Если в процессе оптимизации пропадет звук или перестанет работать сеть:
 * Точка восстановления системы (System Restore): Твой главный спасательный круг. Всегда делай точку перед любым изменением реестра или системных служб.
 * Логирование изменений: Заведи простой текстовый файл dev-machine-log.txt на рабочем столе. Записывай туда каждую отключенную службу или примененный твик. Если через неделю отвалится Wi-Fi, ты будешь точно знать, какое действие это вызвало.

Итоговый манифест оптимизации
Главный принцип
> Не ломай то, что работает, ради копеечной экономии. Борись с тяжелым пользовательским софтом в автозагрузке, а не с системными службами ядра Windows.
> 
Удалить сразу
 * OneDrive, Microsoft Teams, Widgets (Виджеты), Xbox App, Clipchamp, Outlook (новый), Dev Home, To Do, Weather, News, Solitaire.
Отключить сразу
 * Эффекты прозрачности и анимации (Параметры -> Доступность -> Визуальные эффекты).
 * Startup Boost (Быстрый запуск) в настройках Microsoft Edge.
 * Индексацию поиска Windows Search для папки, где будут лежать твои проекты (добавить папку проектов в исключения Defender и Поиска).
Оставить обязательно
 * Windows Defender (Защитник), Брандмауэр (Firewall), SmartScreen, Microsoft Store, Edge WebView2, Snipping Tool (Ножницы), Notepad (Блокнот).
Проверить после недели работы
 * Поведение оперативной памяти при открытых 5 вкладках в Edge и запущенном VS Code. Если свободная память падает ниже 10%, рассмотреть возможность отключения некоторых неиспользуемых расширений VS Code (особенно тяжелых AI-линтеров).
Никогда не делать
 * Не отключать Центр обновлений Windows (Windows Update).
 * Не использовать кастомные сборки Windows (Tiny11/AtlasOS).
 * Не запускать PowerShell-скрипты очистки «в один клик» без построчного анализа их кода.
Минимальная установка FreeX
 * Git для Windows + Windows Terminal (из Microsoft Store).
 * Node.js LTS (установщик .msi с официального сайта).
 * VS Code (устанавливать User Installer, а не System Installer).
