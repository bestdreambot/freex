13.07.2026 — живая память GPT для FreeX

Это не инструкция. Это память, чтобы новый чат мог за несколько секунд
восстановить контекст по команде «Продолжаем». Развёрнутое видение
проекта (философия, идеи по памяти/заметкам/интерфейсу) осталось в
истории git этого файла и в соседних `agents/*.md` — здесь только
фактическое состояние и хронология работы.

## DeepSeek — интеграция (ветка feature/deepseek-env → main)

Что сделано:
- Подключён настоящий DeepSeek API (`https://api.deepseek.com/v1/chat/completions`).
- Приоритет ключей: сначала личный `apiKey` из запроса пользователя,
  иначе `process.env.DEEPSEEK_API_KEY` на сервере.
- Ключ никогда не хранится в коде/Git — только как переменная окружения
  (локально — `.env.local`, защищён `.gitignore`; на Vercel — Environment
  Variables, если настроено).
- Обработка ответа исправлена под реальный формат DeepSeek API:
  `content` и `reasoning_content` — раздельные поля (не смешиваются
  через `</think>`-теги, как было в общем OpenAI-совместимом парсере).
  При наличии `reasoning_content` он возвращается отдельным полем
  `reasoning`, а не вливается в `content`.
- Проверено настоящим запросом к `api.deepseek.com` с реальным ключом —
  ответ `HTTP 200`, содержимое получено.
- Коммит на ветке: `febc138` — `feat: add native DeepSeek provider`.

Что НЕ менялось: GitHub Models, Groq, Cloudflare, Gemini — работают
отдельными ветками в `api/chat.js`, DeepSeek вынесен в свою.

## Аудит архитектуры (перед 8.1)

Полный аудит `index.html`, `style.css`, `vercel.json`, `start.md`,
`api/chat.js`, всех файлов `js/` и `agents/` показал главный факт:

**Двойная архитектура.** Вся боевая логика жила инлайн-скриптом внутри
`index.html` (без `<script type="module">`). Директория `js/`
(app.js, chat.js, notes.js, sidebar.js, ui.js, state.js, db.js,
github.js, utils.js, config.js, api.js) существовала параллельно, но
нигде не подключалась — `js/app.js` был заглушкой-плейсхолдером
(«Рефакторинг в процессе»). Реальный рабочий оркестратор для этой
модульной версии существовал только как текстовый git-патч внутри
`agents/abacus.md` — предложенный, но никогда не применённый.

Дополнительно версия приложения была противоречива: `<title>` и
сайдбар говорили «v6.2», сплэш-экран — «8.0», `start.md` — «5.0».

По итогам аудита Андрей подтвердил: текущая версия работает
неудовлетворительно, и дал разрешение самостоятельно перестроить
`main` напрямую — менять код, реорганизовывать файлы, чинить
архитектуру, коммитить и пушить в `main`, с возможностью отката через
Git.

## Результаты миграции на FreeX 8.1

1. `febc138` (cherry-pick) — `feat: add native DeepSeek provider` —
   DeepSeek-фикс перенесён с `feature/deepseek-env` в `main`.
2. `ab09862` — `refactor: migrate FreeX to modular runtime` —
   инлайн-скрипт в `index.html` полностью удалён, единственная точка
   входа — `js/app.js`. По пути исправлены реальные баги, найденные
   при сверке модулей с рабочим кодом: `js/db.js` импортировал Dexie
   из UMD-сборки без ESM-экспорта (упало бы при импорте), `js/chat.js`
   дважды импортировал `renderSidebarList` (SyntaxError), в
   `js/sidebar.js` был обработчик `onclick` с пропущенной открывающей
   скобкой (клик по чату в сайдбаре не работал). DeepSeek теперь
   доступен без личного ключа и на клиенте — сервер сам подставляет
   `DEEPSEEK_API_KEY`, если пользователь не ввёл свой.
3. `be04faf` — `fix: stabilize chats and conference` — таймаут 55с на
   запрос к модели (`js/api.js`), чтобы Конференция не могла зависнуть
   навсегда на одном провайдере.
4. `c30b078` — `fix: protect notes and drafts` — автосохранение
   черновика заметки, восстановление при повторном открытии,
   предупреждение перед потерей несохранённого, toast-уведомления
   вместо части `alert()`.
5. Документационный коммит (этот файл + `start.md` + `README.md`) —
   версия синхронизирована на FreeX 8.1 везде (было три противоречивых
   номера — теперь один).

Проверки: `node --check` на всех `.js`-файлах (`api/chat.js` +
все `js/*.js`) — без ошибок. Ручной перекрёстный обход показал:
каждый `onclick`/`onchange`/`oninput`/`onkeydown` в `index.html` и
каждый динамически генерируемый обработчик в JS имеют соответствующий
`window.*`-экспорт; все `import` в `js/` разрешаются в реально
существующие экспорты; все провайдеры клиента (`deepseek`, `github`,
`gemini`, `groq`, `cloudflare`) имеют серверные маршруты в
`api/chat.js`. В `git diff` относительно точки возврата секретов не
найдено. Живого браузера/телефона для сквозного теста в этой сессии
не было — это осталось на ручную проверку Андрея.

## Что НЕ менялось / не трогалось

- GitHub Models, Groq, Cloudflare, Gemini — ветки провайдеров в
  `api/chat.js` не изменены.
- Философия Конференции (роли агентов, «дебаты») — оставлена простой,
  как и просил Андрей: сначала надёжность текущей версии, сложные
  роли — отдельным шагом.
- Данные пользователя (localStorage, Dexie/IndexedDB) — схема не
  менялась, миграций не выполнялось, ничего не удалялось.
- Хранение ключей осталось в localStorage в открытом виде (известный
  риск, зафиксирован в `start.md`, шифрование — в бэклоге, не в 8.1).

## Главное правило

Перед любым изменением — прочитать файл целиком, понять архитектуру,
не переписывать с нуля без необходимости. Это подтверждено на
практике в этой сессии: баги в `js/` нашлись и были исправлены именно
потому, что код сверялся построчно с рабочей версией в `index.html` и
с патчем в `agents/abacus.md`, а не переписывался по памяти.
