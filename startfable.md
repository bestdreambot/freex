## Порядок сборки — фазы

**Фаза 0 — фундамент. ГОТОВО.** Правила notion.md, структура HQ, Archive.

**Фаза 1 — один сайт, одна правда. ← ТЫ ЗДЕСЬ**
Свести index.html к одной версии: интерфейс в стиле чата Claude (v0.2) + уже работающее подключение моделей.
Результат фазы: один файл в репо; открывается с телефона; хотя бы одна модель отвечает вживую.

**Фаза 2 — сохранность данных.**
Кнопка «Экспорт»: все чаты и папки в JSON и .md.
Причина: localStorage стирается при очистке кэша — сейчас это главный риск потерять всё. Пока нет экспорта, серьёзно наполнять FreeX данными нельзя.

**Фаза 3 — мульти-агент.**
Параллельные ответы от 2+ моделей с подписями — твой «совет директоров». Добавлять по одной модели за шаг, не все сразу.

**Фаза 4 — заметки.**
Режим заметок рядом с чатами (мост к Obsidian): markdown-заметки, общие папки с чатами.

**Фаза 5 — синхронизация.**
Экспорт/автосинк с GitHub (GitSync или Termux): заметки и чаты живут в репозитории, а не только в браузере.

**Фаза 6 — витрина.**
Публичный start.md как документация, красивая главная, возможно PWA (офлайн).

**Отдельные треки — не смешивать с FreeX:**
- RichTaki — тренировки по своему плану и дневнику
- Best — контент своим ритмом
- Финансы — обзор готов, действия отдельным решением

## Правила работы (сжатая версия notion.md)

1. Время из часов; ответ начинается с даты
2. Черновик → «ок» → сохранение → запись в Archive
3. Только запрошенное, лаконично; один уточняющий вопрос максимум
4. Честность: не говорить «сделано», если не сделано
5. Одно направление за раз. Новые идеи — в Inbox («Входящие данные»), не в работу

## Критерии успеха FreeX (3–6 месяцев)

- Сайт открывается на телефоне и используется каждый день
- Данные не теряются: экспорт работает, копии в GitHub
- Минимум 2 модели отвечают параллельно
- Заметки ведутся в FreeX, а не разбросаны по чатам разных ИИ
- Notion остаётся системой: правила, проекты, архив — без дублей

## Чего НЕ делать сейчас

- Не начинать новые под-проекты внутри FreeX, пока фазы 1–2 не закрыты
- Не держать две версии index.html
- Не хранить API-ключи нигде, кроме localStorage; не коммитить их в GitHub (урок Bybit Hub: засвеченные ключи — менять)
- Не строить сложное (граф связей, роутинг моделей, PWA) до стабильной базы
- Не полагаться вдолгую на `:free`-модели OpenRouter — это промо, может отключиться
