# comp.md — паспорт рабочей машины FreeX

Обновлено: 13.07.2026 · Автор: Андрей

Не инструкция «как ускорить Windows». Это паспорт конкретного ноутбука,
который используется как **FreeX Development Machine**. Правило простое:
**любая программа на этой машине должна уметь ответить на вопрос «зачем она
нужна FreeX?».** Нет ответа — не устанавливается (или не переустанавливается
при восстановлении).

Отвечает на один вопрос: «На какой машине живёт FreeX и как её восстановить?»
Не пересекается с `start.md` (что такое FreeX) и `agent.md` (что мы поняли
в процессе). Здесь — только среда.

---

## 1. Что это

Ноутбук ASUS VivoBook X512FL, используется как основная рабочая станция для
разработки FreeX (VS Code + Claude Code + Git + браузер). Не игровая машина,
не универсальный домашний ПК — рабочий инструмент с одной главной задачей.

## 3. Почему именно она

Это личный ноутбук Андрея, не выделенный сервер — апгрейд RAM пока
финансово не приоритет (см. `PROJECT_STATUS.md`-подобный список purchases
ниже, раздел 13 в истории аудита чата). Работаем с тем, что есть, оптимизируя
софт вместо железа до момента, когда апгрейд станет доступен.

## 5. Что запрещено устанавливать

Любая программа, которая:
- не отвечает на вопрос «зачем это FreeX?» и не относится к отдельным
  личным трекам (Финансы/заметки), явно обозначенным в `start.md`;
- требует постоянно работающей фоновой службы без явной пользы (лишние
  «апдейтеры», трей-приложения, тулбары);
- дублирует уже решённую задачу (второй браузер, второй AI-code инструмент,
  второй антивирус).

Перед установкой чего-либо нового на эту машину — сначала спросить себя
этот вопрос, а не устанавливать по привычке/рекламе.

## 6. Настройки

### BIOS
- Secure Boot и TPM — включены (обязательно для Windows 11, проверять после
  любого обновления BIOS).
- Версия на момент паспорта: X512FLC.301 (2019) — обновление не выполнено,
  проверить сайт ASUS перед следующим большим этапом работы.

### Windows — автозагрузка/службы
- ❌ Из автозагрузки: Movavi Suite Platform, автозапуск Edge.
- ⭐ Служба `gupdate` (Google Update) → Manual (даёт 30с таймаут при загрузке).
- ⭐ Планировщик: отключить `RunPlatformExperienceHelper_*` (Google) и
  `SoftLanding*` (реклама Windows) — не относятся к работе.
- ✔ Visual Effects → «Наилучшее быстродействие» (8 ГБ RAM, каждый мегабайт важен).
- Fast Startup (Hiberboot) сейчас включён — перед обновлением
  BIOS/GPU-драйверов выключить, чтобы не тащить состояние между «выключениями».

### Git
- Глобальный `user.name`/`user.email` не задан — идентичность настроена
  **локально на уровне репозитория** (осознанное решение, см. `agents/gpt.md`).
- `core.autocrlf` не задан явно нигде — источник предупреждений
  `LF will be replaced by CRLF` при коммитах. Задать явно при восстановлении
  (или `.gitattributes` с `* text=auto eol=lf`).
- Авторизация к GitHub — через Windows Credential Manager (HTTPS), **не SSH**
  (папки `~/.ssh` на машине нет — это нормально, просто другой способ).
- **Правило после инцидента с токеном**: никогда не хранить токен встроенным
  в URL remote (`https://oauth2:TOKEN@github.com/...`). Только через
  Credential Manager / `gh auth login`. Проверять `git remote -v` в новых
  клонах перед тем, как ими пользоваться.

### VS Code
- Расширения: только `anthropic.claude-code` + `ms-vscode.powershell` —
  минимум, ничего лишнего, не трогать.
- `settings.json` почти пуст — при восстановлении добавить
  `files.watcherExclude` (node_modules/.git) и `telemetry.telemetryLevel: off`.

### Claude Code
- Одна активная сессия за раз на этой машине (RAM ограничена).
- Новый чат под новую задачу — не наращивать один диалог бесконечно.
- `scripts/checkpoint-freex.ps1` — обязательный шаг перед коммитом, уже
  встроен в процесс.

### Chrome
- Один браузер вместо нескольких (Chrome), Memory Saver включить
  (`chrome://settings/performance`).

### PowerShell
- Все скрипты FreeX в `scripts/` запускаются через
  `-ExecutionPolicy Bypass -File` (не меняет системную политику глобально).

### SSH
- Не используется для GitHub на этой машине — авторизация через HTTPS +
  Credential Manager. Если понадобится SSH в будущем — генерировать новый
  ключ, не переносить старый с других машин.

## 7. Резервные копии

- Код FreeX — в GitHub (`bestdreambot/freex`), это основная защита.
- Точка отката существует: ветка `backup/before-freex-8.1` + тег
  `v8.0-before-rebuild`.
- Windows File History — выключена (`Stopped`). Личные файлы вне git
  сейчас не бэкапятся автоматически.
- Известная проблема на момент паспорта: в **внешней** папке
  `C:\Users\Андрей\Freex\` (отдельно от рабочей `C:\Users\Андрей\Freex\freex\`)
  найден устаревший клон того же репозитория со встроенным токеном GitHub
  в URL remote — токен нужно отозвать на GitHub, сама папка — кандидат на
  удаление при восстановлении. Отложено по решению Андрея до этапа
  форматирования (см. раздел 9).
- Ещё найдены дубли/черновики вне рабочего репозитория: пустые
  Obsidian-vault'ы `Documents\Freex\` и `Documents\Core\` (только конфиг,
  без заметок), и несвязанный локальный черновик кода без git-истории
  `Documents\Freex 2\`. Подробности и рекомендации — в
  `backup/PERSONAL_FILES_CHECKLIST.md`.

## 8. Аккаунты и сервисы, нужные для FreeX

Без значений — только что это и зачем. Сами пароли/ключи — в менеджере
паролей, не в этом файле и не в git.

| Сервис | Зачем FreeX |
|---|---|
| GitHub (`bestdreambot`) | репозиторий, деплой-триггер для Vercel |
| Vercel | хостинг сайта (`freex-eta.vercel.app`) + серверная функция `api/chat.js` |
| Notion | HQ проекта, память и правила (`start.md` §2) |
| DeepSeek API (или свой ключ) | провайдер модели, ключ — в Vercel Environment Variables, не локально |
| Providers из `js/config.js` (GitHub Models, Gemini, Groq, Cloudflare) | опциональные личные ключи пользователя FreeX, хранятся в localStorage браузера |

## 9. План восстановления (кратко — подробности будут в отдельном `INSTALL.md`)

Полный пошаговый install-план (BIOS → Windows → драйверы → Git → Node →
VS Code → Claude Code → Chrome → FreeX → GitHub → проверка → бэкап) —
отдельный документ, будет создан перед фактическим форматированием, не сейчас.

Уже известные шаги, которые обязательно войдут в тот план:
1. Перед форматированием — отозвать токен на GitHub (см. раздел 7).
2. Перед форматированием — свежий `git push` из рабочей папки
   `C:\Users\Андрей\Freex\freex\`, проверить `git status` чист.
3. После чистой Windows — ставить только из списка «✔ Обязательно» (раздел 4),
   остальное — по факту реальной необходимости, не «на всякий случай».
4. После установки — клонировать `bestdreambot/freex` **один раз**, в одну
   папку, без дублей.
5. Применить настройки из раздела 6 (autocrlf, VS Code settings, Visual
   Effects, автозагрузка).
6. Проверка готовности — раздел 10 ниже.

## 10. Как проверить, что всё готово

- [ ] `git --version`, `node --version`, `npm --version` — выводят версии без ошибок
- [ ] `code --list-extensions` — только нужные расширения
- [ ] `powershell -File scripts/resume-freex.ps1` в клонированном `freex` —
      отрабатывает без ошибок, видит git, версию, документы
- [ ] `powershell -File scripts/checkpoint-freex.ps1` — `node --check` проходит,
      секретов не найдено
- [ ] `git remote -v` в клонированном репозитории — URL **без** встроенного токена
- [ ] Vercel деплой доступен по адресу проекта
- [ ] Ни одной программы из списка «❌ Не нужны» не установлено
