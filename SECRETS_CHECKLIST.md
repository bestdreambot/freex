# SECRETS_CHECKLIST.md

Только имена секретов, их назначение и **где искать**. Никаких значений
здесь нет и быть не должно — этот файл идёт в GitHub вместе с остальной
документацией. Сами значения — только в менеджере паролей / там, где они
уже легитимно хранятся (Vercel, браузер, RustDesk).

| Секрет | Для чего | Где хранится сейчас | Куда сохранить перед форматированием | Как вернуть после |
|---|---|---|---|---|
| `DEEPSEEK_API_KEY` | сервер `api/chat.js` использует его, если пользователь не ввёл свой | Vercel Environment Variables (Production) | **никуда — он на сервере Vercel, не на компьютере, форматирование его не тронет** | ничего не нужно, уже там |
| GitHub-токен (для `git`/`gh`) | авторизация push/pull к `bestdreambot/freex` | Windows Credential Manager (`git:https://github.com`) | не нужно сохранять — сгенерировать новый после установки | `gh auth login` или Git Credential Manager при первом push |
| GitHub OAuth-токен (обнаруженный, `gho_...`) | **найден встроенным в URL remote** внешней папки `C:\Users\Андрей\Freex\` (не рабочего репозитория) | буквально в тексте `.git/config` той папки — небезопасно | **отозвать на GitHub до форматирования**, не переносить никуда | новый токен создастся сам через `gh auth login`, если вообще понадобится |
| Персональные ключи провайдеров (GitHub Models, Gemini, Groq, Cloudflare) | ввод пользователя FreeX для дополнительных моделей | `localStorage` браузера на этом компьютере (см. `js/config.js` → `KEY_FIELDS`) | если пользуешься — переписать вручную в заметку в менеджере паролей **до** форматирования, иначе пропадут вместе с профилем браузера | ввести заново в Настройках FreeX после первого запуска |
| Chrome-логин/пароли сайтов | синхронизация браузера | Google-аккаунт (Chrome Sync) | ничего не нужно, если Sync включён | автоматически после входа в Google-аккаунт на новом Chrome |
| RustDesk ID + пароль (unattended access) | удалённый доступ с телефона | `%APPDATA%\RustDesk\config\*.toml` на этом компьютере | **не в GitHub.** Если нужен именно этот ID — вручную переписать значения в менеджер паролей до форматирования | RustDesk сгенерирует новые ID/пароль при переустановке — можно просто настроить заново, это 2 минуты |
| SSH-ключи | — | **не используются на этой машине** (`~/.ssh` не существует, авторизация к GitHub через HTTPS + Credential Manager) | нечего сохранять | если понадобятся в будущем — сгенерировать новые, не переносить со старых бэкапов |
| Vercel-аккаунт (доступ к проекту) | управление деплоем, чтение/правка Environment Variables | сессия в браузере / `vercel login` | ничего не нужно сохранять локально | `vercel login` заново на новой машине, если будет нужен CLI-доступ |
| Notion (HQ проекта) | `start.md` §2, память и правила проекта | сессия в браузере / Notion-аккаунт | ничего не нужно сохранять локально | войти в аккаунт заново |

## Если секрет найден в обычном файле проекта

На момент составления этого чек-листа полный скан всех 39 отслеживаемых
файлов репозитория (`git ls-files`) на паттерны `sk-`, `API_KEY=`,
`Bearer <значение>`, `ghp_/gho_/github_pat_`, `password=`,
`-----BEGIN...PRIVATE KEY-----` — **ничего не найдено**. `.env.local`
существует на диске, но всегда был в `.gitignore` и никогда не
коммитился (`git ls-files | grep env` — пусто).

Единственная найденная за всё время утечка — токен `gho_...` в
`.git/config` **вне** рабочего репозитория (см. таблицу выше) — не в
файле проекта, а в конфиге отдельного, неиспользуемого клона.
