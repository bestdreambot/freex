## 1. Что такое FreeX
Личная мобильная среда для работы с несколькими ИИ сразу, всё с телефона. Один запрос → «Конференция»: все доступные модели отвечают по очереди, каждая читает предыдущих. Гибрид Notion + Obsidian + общего хаба ИИ.

## 3. Архитектура (актуально на 8.1)

**Один рабочий путь, без дублей:**
- `index.html` — только HTML-разметка, подключает `<script type="module" src="js/app.js">`. Инлайновой логики нет.
- `js/app.js` — точка входа: инициализация состояния, обработчики событий, `sendMessage`, обёртка деплоя на GitHub.
- `js/config.js` — модели, ключи localStorage, версия приложения (`APP_VERSION`).
- `js/state.js` — единое состояние приложения (простой event-based store).
- `js/db.js` — Dexie/IndexedDB (заметки), с защитой от недоступности IndexedDB.
- `js/api.js` — вызов `/api/chat` с таймаутом на запрос.
- `js/chat.js` — чаты, Конференция, рендер сообщений, HQ/GitHub-вьюхи.
- `js/notes.js` — заметки: CRUD, поиск, теги, автосохранение черновика.
- `js/sidebar.js` — папки и список чатов.
- `js/github.js` — деплой файлов в репозиторий через GitHub API.
- `js/ui.js` — темы, sidebar toggle, toast-уведомления.
- `js/utils.js` — общие хелперы (localStorage, escapeHtml и т.д.).
- `api/chat.js` — серверный прокси всех провайдеров (Vercel Function).

До версии 8.1 существовала двойная архитектура: рабочий код лежал инлайном в `index.html`, а модульная версия в `js/` была не подключена. Это устранено — единственная точка входа теперь `js/app.js`.

- ✅ Llama 3.3 70B (Groq)

**Интерфейс и функции:**
- Чат: пузыри с аватарами, typing с именем модели, «Копировать» под ответом, история в localStorage, темы (тёмная/светлая/системная), иконка 🔮 на экране iPhone
- **Папки для чатов** — создание, удаление, перенос чата в папку
- **Режим «Конференция»** — все доступные модели отвечают по очереди, с подписями и разделителем; ошибка одной модели не останавливает остальных; таймаут 55с на модель
- **Заметки** — Dexie/IndexedDB, поиск, авто-теги по `#слово`, AI Memory (модель может создавать/обновлять заметки через `<note>`-теги), автосохранение черновика редактируемой заметки с предупреждением перед потерей несохранённого
- **Умный деплой на GitHub** — выпадающий список файлов + кнопка «Очистить поле»
- **HQ (Notion)** — опубликованная страница через iframe
- **Прокси `api/chat.js`** на Vercel — решает CORS, роутит все модели, чистит ответ R1 от рассуждений, ограничивает историю 12 сообщениями
- Экспорт/импорт чатов и папок в JSON (ключи в экспорт не попадают)

## 4. Хранение данных и ключи
- Чаты, папки, ключи, тема, черновики — `localStorage`.
- Заметки — Dexie (IndexedDB), таблица `notes`.
- API-ключи и GitHub-токен хранятся в открытом виде в `localStorage` браузера — не шифруются (известный компромисс v8.1, см. риски ниже), не логируются, не попадают в экспорт.
- `DEEPSEEK_API_KEY` — переменная окружения на Vercel (Production), никогда не хранится в коде/git.

---

## 5. Известные риски v8.1 (не устранены в этой сборке)
- Ключи в `localStorage` — не зашифрованы. При компрометации браузера/устройства ключи читаемы.
- GitHub-токен используется прямо из браузера для вызовов GitHub API — тот же риск хранения.
- Нет автоматических тестов (unit/e2e) — только `node --check` и ручная проверка по чек-листу.
- Мобильное определение Enter-to-send через `navigator.userAgent` — по конструкции хрупко для нестандартных браузеров/PWA-обёрток.
