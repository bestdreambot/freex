## Что такое FreeX

Личная мобильная среда для работы с ИИ, всё с телефона. Два дома:

1. **Notion — Andrey Kyzov's Space HQ** — система и память: правила (notion.md), разделы Me / Settings / Work / Projects / Agents / Inbox / Archive. Здесь порядок.
2. **Сайт FreeX** — GitHub `bestdreambot/freex` → Vercel `freex-xi.vercel.app`. Мобильный ИИ-стенд: чат, история, модели. Здесь инструмент.

Правило номер один: выбери ОДНО, доведи до результата, потом добавляй следующее. Распыление = ноль.

## Что реально работает сейчас (честный статус)

- Единый `index.html` в репозитории, автодеплой на Vercel при каждом коммите
- Чат-интерфейс: пузыри, typing-индикатор, нижний composer, выезжающий sidebar, история в localStorage
- **Две модели: DeepSeek и Grok (xAI).** Ключи вводятся в Настройках, хранятся только в браузере (localStorage)
- **Режим «Дуо»** — на один вопрос отвечают по очереди: сначала DeepSeek, потом Grok, с подписями и разделителем
- **Вкладка «HQ (Notion)»** — показывает опубликованную страницу Notion через iframe
- **Вкладка «Деплой на GitHub»** — обновляет файлы репозитория прямо с сайта через GitHub API (нужен PAT в Настройках). Grok как посредник больше не нужен
- Иконка 🔮 при добавлении сайта на экран iPhone

## Известные ограничения

- Ключи и GitHub-токен лежат в localStorage открытым текстом — ок для личного использования, но при желании усилить — выносить вызовы на serverless-прокси Vercel
- Прямые вызовы моделей из браузера иногда упираются в CORS («Load failed») — если модель перестаёт отвечать, причина чаще тут, а не в ключе. Решение на будущее — прокси на Vercel
- localStorage стирается при очистке кэша браузера — поэтому Фаза 2 (экспорт) важна, пока не наполнять сайт критичными данными

## Порядок сборки — фазы

**Фаза 0 — фундамент.** ГОТОВО. Правила notion.md, структура HQ, Archive.

**Фаза 1 — один сайт, одна правда.** ГОТОВО (07.07.2026). Единый index.html, деплой с сайта работает, DeepSeek и Grok отвечают вживую, дуо по очереди.

**Фаза 2 — сохранность данных.** ← ТЫ ЗДЕСЬ. Кнопка «Экспорт»: все чаты в JSON и .md, чтобы не потерять при очистке кэша.

**Фаза 3 — мульти-агент.** Расширять совет моделей аккуратно, по одной за шаг. Возможно, параллельные ответы.

**Фаза 4 — заметки.** Режим заметок рядом с чатами (мост к Obsidian): markdown, общие папки.

**Фаза 5 — синхронизация.** Чаты и заметки живут в репозитории, а не только в браузере.

**Фаза 6 — витрина.** Публичная документация, красивая главная, возможно PWA (офлайн).

**Отдельные треки — не смешивать с FreeX:** RichTaki (тренировки), Best (контент), Финансы (обзор готов).

## Правила работы

1. Время из часов; ответ начинается с даты
2. Черновик → «ок» → сохранение → запись в Archive
3. Только запрошенное, лаконично; один уточняющий вопрос максимум
4. Честность: не говорить «сделано», если не сделано
5. Одно направление за раз. Новые идеи — в Inbox, не в работу
