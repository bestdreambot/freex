# AI_RESUME — восстановление контекста для любого ИИ-агента

Перед работой прочитай `start.md`, `PROJECT_STATUS.md`, `CHANGELOG.md` и свой
файл в `agents/`. Затем проверь Git (`git status`, `git log -5`). Не изменяй
код до восстановления фактического состояния проекта. Готовый стартовый
запрос — в `START_PROMPT.md`.

Этот файл — журнал восстановления после обрыва связи/сессии. Он обновляется
ДО начала шага (`status: running` + план) и СРАЗУ ПОСЛЕ его завершения
(`status: completed` + факты). Если сессия оборвалась во время шага, статус
останется `running`/`interrupted` — следующий агент обязан сначала проверить
фактическое состояние файлов и только потом продолжать с этого места, не
начиная задачу заново.

## Команда «ок»

Если Андрей присылает сообщение, которое после удаления пробелов по краям
равно (без учёта регистра) `ок`, `OK`, `Ok` или `okay` — это команда на
автоматическое продолжение прерванной работы. Порядок действий агента:

1. Сначала ничего не менять.
2. Прочитать: этот файл, `PROJECT_STATUS.md`, `START_PROMPT.md`, `git status`,
   последние 5 коммитов, список изменённых файлов.
3. Определить из блока «Текущий шаг» ниже: `task`, `step`, `status`, `files`,
   `checks`, `next_step`.
4. Если `status: completed` — этот шаг не повторять, выполнять `next_step`.
5. Если `status: running` или `interrupted` — сверить с фактическим
   состоянием файлов (действительно ли шаг выполнен) и продолжить именно с
   этого места, не начиная задачу заново.
6. Перед началом нового шага — записать `status: running` и план сюда.
7. После успешного завершения — записать `status: completed`, обновить
   `files` / `checks` / `next_step` / `updated_at`.
8. Если текущая задача полностью завершена — не выдумывать новую, сообщить
   «Текущая задача завершена», показать следующий записанный приоритет и
   ждать отдельной команды.

«ок» НЕ даёт разрешения на: удаление пользовательских данных, `git reset`,
force push, раскрытие секретов, установку неизвестных программ, изменение
системных настроек Windows. Разрешены обычные безопасные действия уже
утверждённой задачи: чтение и правка файлов проекта, проверки, создание
вспомогательных файлов, обычный commit/push — но только если он уже был явно
разрешён в текущей задаче.

## Важные решения
- Автоматический блок (`<!-- AUTO:START -->` … `<!-- AUTO:END -->`) в
  `PROJECT_STATUS.md` и в этом файле обновляют только скрипты
  (`resume-freex.ps1`, `checkpoint-freex.ps1`); ручной текст вне этого блока
  скрипты не трогают.
- Поля «Текущий шаг» и «История завершённых шагов» ведёт ИИ-агент вручную —
  их нельзя достоверно вывести из Git.
- Скрипты только читают состояние: никаких commit/push/merge/reset/checkout,
  никогда не выполняют force push или удаление данных.
- Протокол «ок» — это соглашение, которое соблюдает ИИ-агент, читающий этот
  файл; он не подменяет обычные правила безопасности (см. раздел «Команда «ок»»).
